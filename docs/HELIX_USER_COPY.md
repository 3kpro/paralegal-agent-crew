## Error Messages

### API Error
```
⚠️ Connection Error

I'm having trouble connecting to my servers. This usually fixes itself in a moment.

• Check your internet connection
• Try refreshing the page
• If this persists, contact support

[Retry] [Contact Support]
```

### Rate Limit Error
```
⏱️ Slow Down There!

You're sending messages too quickly. Please wait a moment and try again.

(This helps us keep Helix fast for everyone)
```

### Model Timeout
```
⏳ This is Taking Longer Than Expected

Large queries can take time. I'm still working on it...

If this message persists for more than 30 seconds, try:
• Rephrasing your question more specifically
• Breaking complex requests into smaller parts
• Refreshing and trying again
```

## FAQ Responses

**Q: Is Helix free?**
```
Free users get 7 messages per session. Pro users get unlimited access, priority responses, and early access to new Helix features like document upload and conversation history.
```

**Q: What AI powers Helix?**
```
Helix is powered by Google Gemini 2.0 Flash, one of the most advanced language models available. It's fast, accurate, and constantly improving.
```

**Q: Does Helix have access to my data?**
```
Helix can only access data you explicitly share in the conversation or data you've given XELORA permission to access (like your campaigns and analytics). All conversations are encrypted and private to your account.
```

**Q: Can Helix take actions in my account?**
```
Currently, Helix is read-only (view and analyze data only). Workflow automation (having Helix take actions like creating campaigns) is planned for Q3 2026 and will require explicit user confirmation.
```

## Best Practices for Content Writers

1. **Always be honest about limitations:** Don't oversell what Helix can do today
2. **Set clear expectations:** Use "Coming Soon" for roadmap features
3. **Encourage exploration:** Give examples of good questions to ask
4. **Be conversational:** Helix is friendly, not robotic
5. **Provide value:** Even if a feature isn't ready, explain the workaround
